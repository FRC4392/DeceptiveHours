Create a new project using this framework stack:

- Bun as the package manager and script runner
- React Router 7 in SPA mode
- Vite
- React
- TypeScript with strict mode
- Tailwind CSS v4
- shadcn/ui: must use baseUI, not Radix
- CSS variables for theming
- Lucide icons
- Convex for backend/domain data, live queries, mutations, and persistence
- Zustand only for ephemeral client UI state
- next-themes for light/dark/system theme mode
- shadcn Sonner for toast notifications

Use these setup defaults:

- Use Bun commands consistently: `bun`, `bunx`, and `bun run`.
- Use React Router 7 for routing, layouts, navigation, and route organization.
- Use Vite as the frontend build/dev server.
- Use shadcn components wherever possible.
- Do not hand-roll raw form controls when a shadcn component exists.
- Add Sonner through shadcn, not as a custom toast wrapper.
- Use next-themes with class-based theme switching for shadcn dark mode.
- Convex is the source of truth for persisted application/domain data.
- Do not duplicate Convex query data into Zustand.
- Do not create API routes or extra backend layers for data that belongs in Convex.
- Use Zustand only for client-only UI state such as selected item, active tab, panel/sidebar state, dialog state, drag state, temporary editor state, local UI preferences, and other non-domain interaction state.
- Keep TypeScript strict.
- Keep the initial implementation framework-only unless product requirements are provided.

Recommended setup commands:

`bun create vite PROJECT_NAME --template react-ts`
`cd PROJECT_NAME`
`bun add react-router convex zustand next-themes lucide-react sonner`
`bun add -d tailwindcss @tailwindcss/vite`
`bunx shadcn@latest init`
`bunx shadcn@latest add sonner button dialog input textarea select checkbox label dropdown-menu tabs sheet tooltip separator scroll-area`
`bunx convex dev`

After setup, wire the framework baseline only:

- Configure React Router 7 as the app routing layer.
- Configure Vite for React, TypeScript, Tailwind CSS v4, and path aliases.
- Configure Convex provider at the app root.
- Configure next-themes provider using `attribute="class"`, `defaultTheme="system"`, `enableSystem`, and `disableTransitionOnChange`.
- Add the shadcn Sonner Toaster to the root app layout.
- Create a small Zustand store reserved for ephemeral UI state.
- Add or preserve scripts for `dev`, `build`, `lint`, and `typecheck`.
- Do not add product-specific features until separate product instructions are provided.
